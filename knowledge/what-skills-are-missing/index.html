What skills are missing?

Skill gaps become visible when desired use cases are connected to the processes, tools and responsibilities needed to deliver them.

Move beyond software lists

Knowing Revit, Solibri or a CAFM system is not the same as owning an information process. Capabilities include requirement definition, data governance, validation, integration, change management and operational acceptance.

Desired use case→Required process→Responsibility→Skill→Gap

Typical gaps

  • Translating owner goals into structured information requirements.
  • Mapping IFC/IDS rules to project and FM needs.
  • Managing identifiers, classifications and data quality.
  • Connecting BIM delivery with CAFM and portfolio systems.
  • Explaining technical results to decision-makers.
Management use: The same graph can support recruitment, training plans, external sourcing and role design.

A more defensible staffing decision

Instead of asking “Do we need another BIM person?”, the organisation can ask which selected use cases are currently unsupported and which capability would close the gap.
